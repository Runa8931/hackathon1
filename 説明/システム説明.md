# 同期演奏システム 説明

## システム概要

可変抵抗器で設定したBPMに合わせて、複数の楽器Arduinoが同時に演奏するシステム。  
メトロノーム（親機）が拍のタイミングを全楽器側に一斉送信することで同期を実現する。

---

## 全体の信号の流れ

```
可変抵抗器 (A0)
     ↓
metronome.ino（メトロノーム Arduino）
     ↓  UART Serial1  BPM値 or 0（停止）
jyusin.ino（楽器側 Arduino）
     ↓  USB Serial    1（拍）or 0（停止）
guitar_processing_3.pde（Processing / PC）
     ↓
音声再生 + 波形表示
```

---

## 各ファイルの詳細説明

### 1. metronome/metronome.ino — メトロノーム親機

**役割：** 可変抵抗器の値を読み取り、BPMを計算して拍のタイミングで信号を送信する。

#### 主な変数

| 変数 | 説明 |
|------|------|
| `POT_PIN = A0` | 可変抵抗器の接続ピン |
| `BPM_MIN = 20`, `BPM_MAX = 80` | BPMの範囲 |
| `STOP_LEVEL = 10` | この値以下のつまみ位置を「停止」と判断 |
| `beatInterval` | 1拍あたりの周期（ms）= 60000 / BPM |
| `lastBeatTime` | 直近の拍送信時刻（ドリフト防止に使用） |
| `playing` | 演奏中かどうかのフラグ |

#### 関数

**`setup()`**
- `Serial1`（UART）を9600bpsで開始
- `lastBeatTime` を起動時刻で初期化

**`loop()`**
1. 可変抵抗器の値を読む
2. `STOP_LEVEL` 以下 → 停止中なら `Serial1.println(0)` を1回送信して `return`
3. 停止から復帰した直後は `lastBeatTime = now` にリセット（連続送信防止）
4. `map()` でアナログ値(0〜1023)をBPM(20〜80)に変換
5. `beatInterval = 60000L / bpm` で1拍の周期を算出
6. 経過時間が `beatInterval` を超えたら `Serial1.println(bpm)` で送信
   - `lastBeatTime += beatInterval` で次の目標時刻を更新（`= now` にしないことでドリフトを防ぐ）

---

### 2. jyusin/jyusin.ino — 楽器側中継機

**役割：** メトロノームからUARTでBPMを受信し、PCのProcessingへUSBで拍信号を中継する。

#### 主な変数

| 変数 | 説明 |
|------|------|
| `buf[16]` | 受信文字列を蓄えるバッファ |
| `idx` | バッファの書き込み位置 |

#### 関数

**`setup()`**
- `Serial`（USB → PC）を9600bpsで開始
- `Serial1`（UART ← メトロノーム）を9600bpsで開始

**`loop()`**
- `Serial1.available()` で受信データがある間、1文字ずつ読み込む
- 改行（`\n` または `\r`）が来たら行確定 → `handleValue()` を呼ぶ
- それ以外の文字は `buf` に蓄積

**`handleValue(value)`**
- `value == 0` → `Serial.println(0)`（停止をProcessingへ通知）
- `value != 0` → `Serial.println(1)`（拍の合図をProcessingへ通知）
- BPM値の詳細は捨て、「拍か停止か」だけに変換して転送する設計

---

### 3. guitar_processing_3/guitar_processing_3.pde — ギター音源（Processing）

**役割：** Arduinoからの拍信号に合わせて楽譜を読み、Karplus-Strongアルゴリズムでギター音を合成・再生する。

#### 主な定数・変数

| 変数/定数 | 説明 |
|-----------|------|
| `BPM = 160` | Processing内部のBPM（テスト再生用） |
| `RELEASE_MS = 800` | 音の余韻時間（ms） |
| `NOTE_COUNT = 52` | 楽譜の音符数 |
| `noteArray[]` | 各音符の周波数（Hz）配列 |
| `beatArray[]` | 各音符の拍の長さ（1.0 = 1拍）配列 |
| `ksBuffer[]` | Karplus-Strongアルゴリズムの遅延バッファ |
| `testPlaying` | テスト再生中かどうかのフラグ |
| `testNoteIndex` | 現在再生中の音符インデックス |

#### 関数

**`setup()`**
- ウィンドウサイズ設定（800×400）
- Minim・AudioOutputの初期化
- シリアル通信初期化（実機使用時はコメントアウトを外す）

**`draw()`**
- 毎フレーム実行
- `drawWaveform()` で波形を描画
- `testPlaying` が true かつ `nextNoteTime` を過ぎていたら `playNote()` を呼ぶ
- 再生後は次の音符へインデックスを進め、`nextNoteTime` を更新

**`keyPressed()`**
- スペースキー：テスト再生の ON/OFF 切り替え
- Rキー：音符インデックスを先頭に戻す

**`serialEvent(port)`**
- Arduinoから受信があるたびに自動で呼ばれる
- `receiveNoteData()` でデータを取得し `playNote()` を呼ぶ
- 受信中はテストモードを OFF にする

**`receiveNoteData(port)`**
- シリアルから `"周波数,拍長"` の形式で1行読み取り
- `split(",")` でパースして `float[]` で返す
- 失敗時は `null` を返す

**`initKarplusStrong(frequency)`**
- 周波数から遅延バッファのサイズを計算：`ksDelay = round(Fs / frequency * 1.6)`
- バッファをランダムノイズ（-1〜1）で初期化
- これが弦を弾いた瞬間の初期励振を模倣する

**`ksNextSample()`**
- Karplus-Strongアルゴリズムの1サンプル計算
- 現在位置と次位置のサンプルを加重平均（0.7 : 0.3）してローパスフィルタを適用
- `0.999` を乗じることで弦の減衰を制御（1.0に近いほど長く鳴る）
- バッファをリングバッファとして循環させる

**`createWaveform(frequency, samples, durationMs)`**
1. `initKarplusStrong()` でバッファ初期化
2. 全サンプルについて：
   - `ksNextSample()` で基本波形を生成
   - 倍音（2〜6倍音）を少量加算してギターらしさを強調
   - 最初の150サンプルに微小なランダムノイズを加え弦の擦れを表現
   - `applyADSR()` でエンベロープを適用
   - `tanh()` でソフトクリッピング（音割れを防ぎつつ温かみを加える）

**`applyADSR(t, attack, decay, sustain, release, total)`**
- 正規化された時刻 `t`（0〜1）に対して音量を返す
- Attack（立ち上がり）→ Decay（減衰）→ Sustain（持続）→ Release（余韻）の4段階

**`playNote(frequency, beatLen)`**
1. `durationMs = 30000 / BPM * beatLen` で発音時間を計算
2. `createWaveform()` で波形配列を生成
3. `AudioSample` に変換して `trigger()` で再生
4. `delay(durationMs)` で次の音符まで待機

**`drawWaveform(output)`**
- `AudioOutput` のバッファをリアルタイムで折れ線グラフとして描画
- 現在の音名・周波数・拍長・再生位置を画面に表示

**`freqToNoteName(freq)`**
- 周波数を音名（C4, F4, G4 ... D5）に変換する表示用ヘルパー

**`stop()`**
- Processingウィンドウを閉じる際にAudioOutputとMinimを安全に終了する

## Karplus-Strongアルゴリズムの仕組み（補足）

```
初期化：バッファをランダムノイズで満たす（弦を弾いた瞬間）
         ↓
ループ：隣接サンプルの加重平均 × 減衰係数 → バッファを更新
         ↓
出力：バッファから1サンプルずつ取り出す
```

ローパスフィルタを繰り返し通すことで高周波成分が先に消え、  
低周波成分が残る → 弦楽器の自然な減衰特性を物理的に模倣している。

---

## テストモードと本番モードの切り替え

| モード | 動作 |
|--------|------|
| **テストモード**（スペースキー） | Processing内部のBPM/楽譜データで自律再生 |
| **本番モード**（シリアル受信時） | Arduinoからの拍信号に合わせて同期再生 |

シリアルから信号を受け取ると自動的にテストモードが OFF になる設計。
